# StreamCap 本地变更记录

## 2026-06-24

### 相对原仓库的本地改动

- 将 `pyproject.toml`、`requirements.txt`、`requirements-web.txt` 中的 `streamget` 依赖改为指向最新的 `ihmily/streamget` GitHub 仓库。
- 修复 Web 模式界面加载时的资源噪声，允许 Flet 继续使用 CDN 字体回退，避免反复请求缺失的本地 `assets/fonts/notosanssc/...` 资源。
- 在 Web 模式下过滤预期内的客户端断开连接堆栈，避免浏览器刷新或关闭标签页时刷屏后端日志。
- 仅为录制房间加入 SQLite 持久化；通用应用设置、Cookie、账号配置、语言配置和 Web 登录认证仍保留 JSON 文件。
- 新增 `data/database/recordings.db`，包含录制房间、录制历史和数据库元信息表。首次加载且数据库为空时，会从旧的 `config/recordings.json` 迁移房间列表。
- 为录制房间新增 `created_at`、`updated_at`、`pinned_at`、`pin_order`、`last_recorded_at`、`last_record_file`、`last_live_title` 等元数据。
- 在 FFmpeg 或直连下载录制完成、停止、失败时写入录制历史。
- 在录制卡片上新增置顶/取消置顶操作，并在录制信息弹窗中显示加入时间、最近录制时间和最近录制文件。
- 在侧边栏新增“日志”页面，用于查看 SQLite 中的最近录制历史。
- 优化置顶功能：增加可见的置顶标签、高亮置顶卡片，并保证刷新、新增和跨会话更新后的卡片排序稳定。
- 修复置顶中的边界问题，避免正在录制的卡片因置顶后的即时重排或相邻按钮误触而意外停止录制。
- 调整录制卡片操作：录制信息改为双击卡片打开，同时保留停止监控按钮，并让置顶按钮样式与其他卡片操作保持一致。
- 将日志页面细分为运行日志、操作日志和结构化录制历史，避免把录制历史重复当作普通日志展示。
- 修复日志/历史记录不可见的问题：恢复 Flet `run`/`app` 启动兼容层，放宽操作日志识别范围，增加日志页刷新诊断，并在 FFmpeg 关闭时即使没有 stderr 输出也记录停止历史。
- 在 `README.md` 中补充 Windows exe 打包、动态 Web 运行和 Flet 静态 Web 构建说明，并标注完整录制服务对 Python 后端、FFmpeg、配置目录、日志和 SQLite 数据库的依赖。
- 完成 Windows exe 打包测试：`flet pack` 可生成桌面端 onedir 产物，补充基础 `config` 文件复制步骤，并明确 Web 服务应使用 `python main.py --web` 或 Docker 动态运行。
- 将本地打包测试产物目录 `dist-pack-test/` 加入忽略，避免测试包进入版本对比。
- 调整置顶按钮图标：未置顶时显示空心图钉，已置顶时显示实心图钉，保持与卡片操作栏的 Material 单图标风格一致。
- 修复日志页在 Flet 0.85.3 下打开失败的问题，将不兼容的 `ft.padding.symmetric` 改为 `ft.Padding.symmetric`。
- 扩展并简化存储页：保留原目录文件浏览，新增存储概览、文件类型统计、`original/` 原始文件统计和最近录制文件；最近录制文件统一使用视频图标；清理工具提前到目录文件前方，并仅保留删除全部文件、清理空目录、清理日志；移除文件校验和备份导出入口。
- 存储概览移除零字节文件指标，减少不必要的信息展示。
- 重新设计存储页布局为单列管理视图：顶部集中显示路径和操作按钮，存储概览优先展示，清理工具独立成行，目录文件恢复为完整主区域，最近录制文件放在目录下方，避免右侧窄栏导致按钮和列表挤压。
- 调整存储页信息密度：合并存储概览指标，取消文件类型统计展示，并将目录文件与最近录制文件改为两栏并列显示。
- 删除存储页已废弃的文件类型统计本地化文案键，减少无用配置残留。
- 修复存储页两栏布局：目录文件与最近录制文件使用一致高度，桌面端按比例并列显示，移动端上下排列，避免 `wrap` 与 `expand` 混用导致目录区域不显示。
- 统一存储概览中合并指标卡片的高度，避免不同指标行数导致卡片高度不一致。
- 调整存储清理文案，将“删除全部文件”改为“删除录制文件”，并同步确认弹窗说明其只清理媒体录制文件。
- 完善日志页：运行/操作日志继续使用自动轮转文件日志，录制历史继续使用 SQLite；日志展示增加中文化解析，运行日志、操作日志和录制历史分别增加清空功能。
- 简化日志页分类：移除操作日志页签，仅保留运行日志和录制历史；录制历史按开始录制与停止录制事件区分展示。
- 调整录制历史展示为单栏事件列表，开始录制和停止录制按时间混排展示，并用绿色/红色边框区分事件类型。
- 修复录制历史记录时机：录制开始时立即写入开始录制事件，录制停止/完成时写入停止事件，并兼容旧的完成记录拆分显示。
- 调整录制历史事件卡片样式：统一灰色边框，开始/停止录制通过绿色/红色图标和文字区分，并固定事件卡片高度以保持列表整齐。
- 优化录制历史开始事件卡片，移除“录制时间：无”“文件地址：无”等无意义占位信息。
- 重新排列录制历史事件卡片：使用图标标签、主播与时间同一行、平台/时长/文件分层展示，减少空白并保持卡片高度整齐。
- 修复存储页和日志页确认弹窗关闭逻辑，确认或取消后先推送 `open=False`，再清空弹窗承载区域，避免遮罩层残留导致界面无法继续操作。
- 将“删除录制文件”由二次确认改为一次确认，保留文件数量和预计释放空间提示，避免嵌套确认弹窗在 Flet 中残留遮罩。
- 修复存储页“最近录制文件”仍显示已删除文件的问题：列表只展示当前仍存在的媒体文件，并在播放入口增加文件存在性检查。
- 对齐录制界面的播放策略：存储页“最近录制文件”在历史文件路径失效时，会回退扫描录制输出目录并选择最新媒体文件用于展示和播放。
- 修复 SQLite 仓库连接释放问题，确保每次数据库操作结束后关闭连接，避免 Windows 下数据库文件被占用。
- 调整运行时数据目录：日志文件写入 `log/`，录制数据库写入 `data/database/`，并兼容迁移旧的 `config/recordings.db`。
- 修复 GitHub Actions 构建流程：移除重复的自动构建 workflow，macOS 打包改用 `.icns` 图标，并使用 `github.token` 更新 beta 标签。
